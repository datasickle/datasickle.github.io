JAVA面向对象学习的三条主线

  1. JAVA类及类的成员：属性、方法、构造器、代码块、内部类
  2. 面向对象的三大特征：封装性，继承性，多态性
  3. 其他关键字：this、super、static、final、abstract、interface、package、import等

1.面向对象思想概述

  • 类(Class)和对象(Object)是面向对象的核心概念

    • 类是对一类事物的描述，是抽象的、概念上的定义

    • 对象是实际存在的该类事物的每个个体，因而也称为实例(instance)

  • “万事万物皆为对象”

    如“人类”就是一个抽象的类，他是所有不同的人类的泛称，是一个概念。

    而具体到某一个人。则就是这个类的实例，如“野兽先辈”。

    • 可以理解为：类 = 抽象概念的人；对象 = 实实在在的某个人

    • 面向对象程序设计的重点是类的设计

    • 类的设计，其实就是类的成员的设计

  • 类的成员

    • 属性：对应类中的成员变量 = Field = 域
    • 行为：对应类中的成员方法 = Method = 函数

实例

我们定义一个人“Person”类。

类中有属性和方法

//如下，可以通过class关键字+类名的方法创建一个类
class Person
{
    //可以在类中定义属性和方法
    /*
    属性（成员变量/field/域/字段）
    人这个类有什么属性呢？应该有姓名，年龄等。
     */
    String name;//定义姓名属性。用数据类型+属性名
    int age;//年龄
    boolean isMale;//是否为男性
    /*
    方法（成员方法/method/函数）。
    即人这个类有什么功能
     */
    public void eat()//会吃饭
    {
        System.out.println("人会吃饭");
    }

    public void sleep()//会睡觉
    {
        System.out.println("人会睡觉");
    }

    public void talk(String language)//会说话。定义局部变量（形参），可以在调用方法的时候传输实参。
    {
        System.out.println("人会说话，使用的是：" + language);
    }
}

2.类的使用

我们有了一个类，那如何使用这个类呢？

这时就要用到实例化。就像“人类”是个广泛的、抽象的概念，而实例化就相当于具体到某个人。

类：人类 这个类中有属性、有方法

实例化对象：张三 张三这个对象拥有类中的一切属性和方法

/测试类
public class OOPTest
{
    public static void main(String[] args)
    {
        //实例化Person类的对象（对象的实例化/instance）
        Person person1 = new Person();//类名 变量名（对象名） = new 类名
        //有了对象，就能调用结构：属性、方法
        //调用属性：“对象名.属性”
        person1.name = "Jack";
        person1.age = 18;
        person1.isMale = true;
        System.out.println("名字：" + person1.name + "\n" + "年龄：" + person1.age);
        //调用方法：“对象名.方法”
        person1.eat();//调用吃的方法
        person1.sleep();//调用睡觉的方法
        person1.talk("English");//调用说话方法，并传参数
    }
}

输出结果

名字：Jack
年龄：18
人会吃饭
人会睡觉
人会说话，使用的是：English

3.一个类的多个对象

此时，我们再实例化一个对象。

没错，一个类可以实例化无穷无尽的对象。就如同人类这个庞大的群体下，有着好几十亿的不同的对象。

“中国人”这个类下面，就有着十几亿的实例。

//测试类
public class OOPTest
{
    public static void main(String[] args)
    {
        //创建Person类的对象（对象的实例化/instance）
        Person person1 = new Person();//类名 变量名（对象名） = new 类名
        //有了对象，就能调用结构：属性、方法
        //调用属性：“对象名.属性”
        person1.name = "Jack";
        person1.age = 18;
        person1.isMale = true;
        System.out.println("名字：" + person1.name + "\n" + "年龄：" + person1.age);
        //调用方法：“对象名.方法”
        person1.eat();//调用吃的方法
        person1.sleep();//调用睡觉的方法
        person1.talk("English");//调用说话方法，并传参数

        System.out.println("=========================================================");

        //再实例化一个对象
        Person person2 = new Person();
        person2.name = "ZhangSan";
        person2.age = 33;
        person2.isMale = true;
        System.out.println("名字：" + person2.name + "\n" + "年龄：" + person2.age);

        //用这个对象调用方法。用什么对象调用方法，那么方法所用到的属性就是这个对象的。
        person2.eat();
        person2.sleep();
        person2.talk("中文");
    }
}

输出结果

名字：Jack
年龄：18
人会吃饭
人会睡觉人会说话，使用的是：English
——————————————————————
名字：ZhangSan
年龄：33
人会吃饭
人会睡觉
人会说话，使用的是：中文

如果创建了一个类的多个对象，则每个对象都独立的拥有一套类的属性（非static的）

意味着：如果我们修改一个对象的属性A，则不影响另外一个对象属性A的值。

即：属性不一样，方法一样，但方法中的变量值不一样。

同样，一个对象可以赋值给另一个对象。如下，我们new一个person3，将person1赋值给person3，其实是将整个对象都赋值给了person3，无论是属性，还是方法。其实是将person1变量保存的对象地址值赋给了person3，导致person1和person3指向了堆空间中的同一个对象实体。 

//new一个person3，将person1赋值给person3
Person person3 = person1;
System.out.println("名字：" + person3.name + "\n" + "年龄：" + person3.age);
person3.eat();
person3.sleep();
person3.talk("Japanese");

输出结果

名字：Jack
年龄：18
人会吃饭
人会睡觉
人会说话，使用的是：Japanese

可以看到的是，person3.talk的参数被修改成了”Japanese”，而person1.talk的参数则是”English”，这说明方法是可以重载覆盖的。

而由于person1和person3指向了堆空间的同一个对象实体，那么修改二者中任意一方的属性数值，另一方也会随之改变。

如下，修改person3的age变量，因为person3和person1指向同一个实体。所以就相当于在修改person1.

Person person3 = person1;
person3.age = 102;
System.out.println("person1的age此时为："+person1.age);
System.out.println("名字：" + person3.name + "\n" + "年龄：" + person3.age);
person3.eat();
person3.sleep();
person3.talk("Japanese");

输出结果

person1的age此时为：102 //此时person3修改了age，person1也随之改变。
名字：Jack
年龄：102
人会吃饭
人会睡觉
人会说话，使用的是：Japanese

4.对象的内存解析

运行时数据区

其中包含了：

  • 方法区(Method Area)
    • 用于存储已被虚拟机加载的类信息、常量、静态变量、即时编译器编译后代码等数据。
  • 虚拟机栈(VM Stack) : 通常所说的栈 特点是：先进后出
    • 虚拟机栈用于存储局部变量等。局部变量表存放了编译期可知长度的各种基本数据类型(boolean、byte、char、short、int、float、long、double)、对象引用(reference类型，它不等同于对象本身，是对象在堆内存的首地址)。方法执行完，自动释放。
  • 本地方法栈(Native Method Stack)
  • 堆(Heap)
    • 此区域唯一目的就是存放对象实例，几乎所有的对象实例都在这里分配内存。这一点在Java虚拟机规范中的描述是：所有的对象实例以及数组都要在堆上分配。
  • 程序计数器(Program Couter Register)

5.类中属性的声明和使用

属性（成员变量）VS 局部变量

  1. 相同点

    1. 定义变量的格式：数据类型 变量名 = 变量值
    2. 先声明，后使用
    3. 变量都有其对应的作用域

  2. 不同点

    1. 在类中声明的位置不同

      • 属性（成员变量）：直接定义在一对{}内

      • 局部变量：声明在方法内、方法形参、代码块内、构造器形参、构造器内部的变量

  3. 关于权限修饰符的不同

    • 属性（成员变量）：可以在申明属性时，指明其权限，使用权限修饰符 常用限修饰符：private、public、缺省、protected ———> 封装性
    • 局部变量：不能使用权限修饰符
  4. 默认初始化值的情况
    • 属性（成员变量）：类的属性，根据其变量类型，都有其初始化值
          整型(byte,short,int,long)：0
    浮点型(float,double)：0.0
    字符型(char)：0（或'\u0000'）
    布尔型(boolean)：false
    引用数据类型(类、数组、接口)：null
    • 局部变量：没有默认初始化值
          意味着，除形参外，我们在调用局部变量之前，一定要显式赋值。
    特别的：形参在调用时，我们赋值即可
      1. 二者在内存中加载的位置也不同
    • 属性（成员变量）：加载到堆空间中（非static的）
    • 局部变量：加载到栈空间

6.1 关于权限修饰符

Java中规定的4种权限修饰符：private、public、缺省、protected

private：私有的

public：公共的

缺省：默认的

protected：受保护的

6.2 返回值类型

有返回值 VS 无返回值

若方法有返回值，则必须在方法声明时，指定返回值的类型。同时，方法中，需要使用return关键字，来返回指定类型的变量或常量。

若方法无返回值，则在方法声明时，使用void来表示。通常，没有返回值的方法中，就不用return。但是，如果使用的话，则直接用”return;”，表示方法结束。return后，无法在其后跟任何表达式及语句。

如何判断什么时候该添加返回值呢？

主要看要求和经验，在定义一个方法（函数）时，如果需要该方法给一个明确的答复，那肯定得需要一个返回值，否则我跟它要东西，他不给我，会很尴尬。

6.3 方法名

方法名：属于标识符，遵循标识符的规则和规范。“见名知意”。

6.4 形参

一个方法可以声明0个、1个或多个形参

格式：数据类型1 形参1,数据类型2,形参2,…

我们定义方法时，该不该定义形参？

看经验。

6.5 方法体

方法功能的体现，即方法实现的过程，方法的具体逻辑代码就在方法体中完成。

6.6 其他

方法的使用中，可以调用当前类的属性或方法。也就是说，方法内部可以调方法，输出的时候一并执行。

特殊的：方法A中又调用了方法A，称为“递归方法”。

方法中，不可以再定义方法。

7.return关键字的使用

  1. 使用范围：使用在方法体中

  2. 作用：① 结束方法

    ​ ② 针对有返回值类型的方法，使用”return 数据;”返回所要的数据。

  3. 注意点：return关键字后面不可以声明执行语句，类似于”break”.
