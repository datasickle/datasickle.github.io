第2章：获取数据库连接

导入驱动包

Driver接口，是位于java.sql.Driver下的一个接口。

每个驱动程序类都必须实现的接口。这些驱动程序类由数据库公司提供，当驱动程序类实现了Driver接口，便可以与数据库进行交互等。

那么如何获得数据库公司所提供的驱动程序呢？这里以MySQL为主。

首先去MySQL官方网站下载驱动：MySQL官网🔍

下载完成后，将压缩包中的jar文件解压

而后复制解压出来的jar文件，在java工程中创建一个lib文件夹，这里用来存放我们需要导入的jar包，而后将jar文件粘贴进去。

右键粘贴好的文件，选择add as library，确认即可。

至此，驱动包导入完毕。我们接下来便可以连接数据库了。

连接方式1

Driver接口下，有个connect方法，当然是需要实现类来调用。

此方法是“试图通过给定的url连接到指定的数据库”。当然，需要指定String url,Properties info这两个形参。此方法返回一个Connection类型的变量值。

步骤：

  1. 我们已经导入了驱动包，所以要new一个Driver接口的实现类，而后用实现类，去调实现类重写后的方法。Driver接口的实现类位于包的com.mysql.cj.jdbc.Driver

  2. 然们用实现类调方法，也就是connect(String url, Properties info)

  3. 调用方法并给形参赋值后，我们返回一个Connection类型的值，返回的是实现类的对象。我们输出这个值，若成功输出，则连接成功。

形参中的两个数据分别是什么？

String url：“jdbc:mysql://localhost:3306/test?serverTimezone=Asia/Shanghai”；

这里面分别是 jdbc:数据库协议://ip:端口/数据库的名字?serverTimezone/=时区(洲/城市)

Properties info : 将用户名和密码封装

public class ConnectionTest
{
    @Test
    public void testConnection1() throws SQLException
    {
        //首先，获取Driver接口的实现类对象
        Driver driver = new com.mysql.cj.jdbc.Driver();
        //而后通过new出来的对象，去调实现类的重载方法。当然得先指定方法所需要的形参
        //此处指定链接
        String url = "jdbc:mysql://localhost:3306/test?serverTimezone=Asia/Shanghai";
        //分别指定用户名和密码
        Properties info = new Properties();
        info.setProperty("user","root");
        info.setProperty("password","123456");
        //而后这里方法接收到形参后，我们返回一个值
        Connection connect = driver.connect(url, info);
        //将值输出，如果成功，则连接数据库成功，如果失败，则失败。
        System.out.println(connect);
    }
}

输出结果：

com.mysql.cj.jdbc.ConnectionImpl@13c9d689

连接成功。

连接方式2

对方式1的迭代

为什么要迭代？因为面向接口编程，需要很好的可移植性，所以代码中不要出现任何的第三方API。

所以我们需要用反射来获取接口的实现类对象。

//方式2：对方式1的迭代，使用反射。
@Test
public void testConnection2() throws Exception
{
    //1.获取Driver实现类对象，使用反射
    Class clazz = Class.forName("com.mysql.cj.jdbc.Driver");
    Driver driver = (Driver) clazz.newInstance();

    //2.提供要连接的数据库
    String url = "jdbc:mysql://localhost:3306/test?serverTimezone=Asia/Shanghai";

    //3.提供连接所需要的用户名和密码
    Properties info = new Properties();
    info.setProperty("user","root");
    info.setProperty("password","200625");

    //4.获取连接
    Connection connect = driver.connect(url, info);
    System.out.println(connect);
}

输出结果：

com.mysql.cj.jdbc.ConnectionImpl@13c9d689

连接方式3

使用DriverManager这个类替代driver。

该类位于java.sql.DriverManager

本类中，提供了一个静态方法，用来注册驱动

提供了如下3个静态方法，可以连接到数据库。

//方式3：使用DriverManager替换Driver
    @Test
    public void testConnection3() throws Exception
    {
        //1.获取Driver实现类对象
        Class clazz = Class.forName("com.mysql.cj.jdbc.Driver");
        Driver driver = (Driver) clazz.newInstance();
        //2.提供所需要的基本参数
        String url = "jdbc:mysql://localhost:3306/test?serverTimezone=Asia/Shanghai";
        String user = "root";
        String password = "200625";
        //3.注册驱动
        DriverManager.registerDriver(driver);
        //4.获取连接
        Connection connection = DriverManager.getConnection(url, user, password);
        System.out.println(connection);
    }

连接方式4

在方式3的基础上做一些优化。

//方式4：对方式3的优化
@Test
public void testConnection4() throws Exception
{
    //1.提供所需要的基本参数
    String url = "jdbc:mysql://localhost:3306/test?serverTimezone=Asia/Shanghai";
    String user = "root";
    String password = "200625";

    //2.加载驱动
    Class.forName("com.mysql.cj.jdbc.Driver");
    //Driver driver = (Driver) clazz.newInstance();
    //注册驱动
    //DriverManager.registerDriver(driver);
    //3.获取连接
    Connection connection = DriverManager.getConnection(url, user, password);
    System.out.println(connection);
}

由于 Class.forName(“com.mysql.cj.jdbc.Driver”);这行代码是将这个类加载到了内存中，而在这个类的源代码当中，存在着一个静态代码块，由于静态代码块随着类的加载而执行，所以无需我们手动注册驱动。

public class Driver extends NonRegisteringDriver implements java.sql.Driver {
    public Driver() throws SQLException {
    }

    static {
        try {
            DriverManager.registerDriver(new Driver());
        } catch (SQLException var1) {
            throw new RuntimeException("Can't register driver!");
        }
    }
}

其实，连第一步的加载都可以省略。

@Test
public void testConnection4() throws Exception
{
    //1.提供所需要的基本参数
    String url = "jdbc:mysql://localhost:3306/test?serverTimezone=Asia/Shanghai";
    String user = "root";
    String password = "200625";

    //2.获取Driver实现类对象
    //Class.forName("com.mysql.cj.jdbc.Driver");
    //Driver driver = (Driver) clazz.newInstance();
    //注册驱动
    //DriverManager.registerDriver(driver);
    //3.获取连接
    Connection connection = DriverManager.getConnection(url, user, password);
    System.out.println(connection);
}

输出结果：

com.mysql.cj.jdbc.ConnectionImpl@13c9d689

这是为什么呢？实际上，就连加载这一步，也自动帮我们做了。为什么呢？

因为在我们导入的包中，有个文件，文件中的地址，就是我们需要加载的类。在我们导入包之后，系统自动就帮我们加载好了。

但是，在实际开发的过程中，第一步的加载最好不要省。

连接方式5

优化配置信息：

如url，user，password等，不要以硬编码的方式写到代码中，而是将这些信息写入配置文件中，通过读取文件的方式来获得信息。

我们在src目录下新建一个jdbc.properties的文件，将基本信息写进去

user=root
password=1234
url=jdbc:mysql://localhost:3306/test?serverTimezone=Asia/Shanghai
driverClass=com.mysql.cj.jdbc.Driver
@Test
//方式5：将基本信息写入配置文件，通过读取文件的信息，获取连接
public void getConnection5() throws Exception
{
    //1.读取配置文件中的4个基本信息.
    //通过类加载器加载
    InputStream rs = ConnectionTest.class.getClassLoader().getResourceAsStream("jdbc.properties");
    Properties ps = new Properties();
    ps.load(rs);
    String user = ps.getProperty("user");
    String password = ps.getProperty("password");
    String url = ps.getProperty("url");
    String driverClass = ps.getProperty("driverClass");

    //2.加载驱动
    Class.forName(driverClass);

    //3.获取连接
    Connection connection = DriverManager.getConnection(url,user,password);
    System.out.println(connection);
}

输出结果：

com.mysql.cj.jdbc.ConnectionImpl@3754a4bf

这样的好处是实现了数据和代码的分离，实现了解耦。
