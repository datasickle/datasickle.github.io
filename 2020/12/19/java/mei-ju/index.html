枚举

JDK5.0之前，枚举类需要自定义。而JDK5.0之后，新增了关键字enum来更方便地定义枚举类。

注：enum为关键字，Enum为类。关键字enum定义的枚举类继承于java.lang.Enum类。而自定义枚举类则继承于Object。

什么是枚举类？

🔴枚举是一个被命名的整型常数的集合, 枚举在日常生活中很常见。枚举类的对象必须是有限个且确定的。

​ 如：

​ 季节类中有4个对象：春天(Spring)，夏天(Summer)，秋天(Autumn)，冬天(Winter)

​ 星期类中有7个对象：星期一（Monday）……星期天(Sunday)

​ 性别类中有2个对象：男(Man)，女(Woman)

​ 等……

🔴当需要定义一组常量时，可以使用枚举类。如：当定义的都是某一个东西的不同状态时，可以使用。常量 意味着不能修改，用枚举类定义后拿来用就行。

🔴如果枚举类中只有一个对象，则可以作为单例模式的实现方式。

如何定义枚举类？

方式一

JDK5.0之前自定义

package com.test.TestEnum;

import org.junit.Test;

public class TestEnum
{
    public static void main(String[] args)
    {
        //枚举类在下面定义好了，接下来直接调用就可以了
        //用类名.对象名调用 并赋值
        Season spring = Season.SPRING;
        Season summer = Season.SUMMER;
        Season autumn = Season.AUTUMN;
        Season winter = Season.WINTER;
        //打印输出
        System.out.println(spring);
        System.out.println(summer);
        System.out.println(autumn);
        System.out.println(winter);
    }

}
//自定义枚举类(JDK5.0之前)
class Season
{
    //1：声明Season对象的属性，私有化的常量，用private final来修饰
    private final String seasonName;//季节名
    private final String seasonDesc;//季节描述

    //2：类的构造器需要私有化(防止外部调用，如果外部能调用，则说明类的对象个数不确定，那就不是枚举了)
    private Season(String seasonName,String seasonDesc)//在构造器中给季节属性赋值
    {
        this.seasonName = seasonName;
        this.seasonDesc = seasonDesc;
    }
    //3：提供当前枚举类的多个对象。
    /*因为上面属性是私有化且不变的常量，所以不提供get,set方法。
      但外部又需要获取枚举类的对象，所以用public修饰
      因为要用类直接调用。所以用static修饰
      同时参数还要是不变的，于是用fianl修饰*/
    public static final Season SPRING = new Season("春天","春风得意马蹄疾");
    public static final Season SUMMER = new Season("夏天","别院深深夏席清");
    public static final Season AUTUMN = new Season("秋天","八月秋高风怒号");
    public static final Season WINTER = new Season("冬天","已是悬崖百丈冰");

    //4：其他需求 如果需要调用类中的属性，则提供get方法
    //注：被final修饰过的属性为常量，所以无法提供set方法

    public String getSeasonName()
    {
        return seasonName;
    }

    public String getSeasonDesc()
    {
        return seasonDesc;
    }

    //5：其他需求 提供toString方法

    @Override
    public String toString()
    {
        return "Season{" +
                "seasonName='" + seasonName + '\'' +
                ", seasonDesc='" + seasonDesc + '\'' +
                '}';
    }
}

输出结果（重写toString后）：

Season{seasonName='春天', seasonDesc='春风得意马蹄疾'}
Season{seasonName='夏天', seasonDesc='别院深深夏席清'}
Season{seasonName='秋天', seasonDesc='八月秋高风怒号'}
Season{seasonName='冬天', seasonDesc='已是悬崖百丈冰'}

方式二

JDK5.0时，新增了enum关键字，可以使用它来定义枚举类

注：使用enum关键字来定义枚举类时，默认继承于java.lang.Enum类

package com.test.TestEnum;
/*使用enum关键字来定义枚举类
    JDK5.0及以后
 */
public class TestEnum1
{
    public static void main(String[] args)
    {
        Season1 spring = Season1.SPRING;
        Season1 summer = Season1.SUMMER;
        Season1 autumn = Season1.AUTUMN;
        Season1 winter = Season1.WINTER;
        System.out.println(spring);
        System.out.println(summer);
        System.out.println(autumn);
        System.out.println(winter);
    }

}
enum Season1
{
    //1：直接提供当前枚举类的对象
    //多个对象之间用逗号隔开，最后一个用分号结尾
     SPRING("春天","春风得意马蹄疾"),
     SUMMER("夏天","别院深深夏席清"),
     AUTUMN("秋天","八月秋高风怒号"),
     WINTER("冬天","已是悬崖百丈冰");
    //2：声明对象的属性
    private final String seasonName;
    private final String seasonDesc;
    //3：私有化构造器，并给属性赋值
    private Season1(String seasonName, String seasonDesc)
    {
        this.seasonName = seasonName;
        this.seasonDesc = seasonDesc;
    }
    //4：提供其他诉求，get方法和覆写toString

    public String getSeasonName()
    {
        return seasonName;
    }

    public String getSeasonDesc()
    {
        return seasonDesc;
    }

    @Override
    public String toString()
    {
        return "Season1{" +
                "seasonName='" + seasonName + '\'' +
                ", seasonDesc='" + seasonDesc + '\'' +
                '}';
    }
}

enum由于继承于java.lang.Enum类，所以它的toString方法本身就被重写过，如果用户自己不重写，则打印的是对象的名字

SPRING
SUMMER
AUTUMN
WINTER

用户重写后则打印

Season1{seasonName='春天', seasonDesc='春风得意马蹄疾'}
Season1{seasonName='夏天', seasonDesc='别院深深夏席清'}
Season1{seasonName='秋天', seasonDesc='八月秋高风怒号'}
Season1{seasonName='冬天', seasonDesc='已是悬崖百丈冰'}

总结：定义枚举类的大致顺序

1️⃣：当enum类定义好后，上来就直接给出类的对象

enum Season
{
    SPRING,SUMMER,AUTUMN,WINTER;
}

2️⃣：而后再去声明属性，属性必须是私有化的常量

private final String seasonName;
private final String seasonDesc;

3️⃣：而后提供私有化的有参构造器，给属性赋值

private Season1(String seasonName, String seasonDesc)
{
    this.seasonName = seasonName;
    this.seasonDesc = seasonDesc;
}

4️⃣：去第一步，给属性设置实参，或者在第一步直接设置好也是一样的。注：参数类型要一致

enum Season
{
	SPRING("春天","春风得意马蹄疾"),
	SUMMER("夏天","别院深深夏席清"),
	AUTUMN("秋天","八月秋高风怒号"),
	WINTER("冬天","已是悬崖百丈冰");
}

5️⃣：提供其他的需求，如get方法和重写toString方法

//提供get方法
public String getSeasonName()
{
    return seasonName;
}

public String getSeasonDesc()
{
    return seasonDesc;
}

//重写toString方法
@Override
public String toString()
{
    return "Season1{" +
            "seasonName='" + seasonName + '\'' +
            ", seasonDesc='" + seasonDesc + '\'' +
            '}';
}
